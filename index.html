Google Sheets Sync
Connect live data · Override in bot anytime
i
How this works: Google Sheets are your source of truth for raw data. Once you publish a tab as a CSV URL, the bot can pull it live. You can always override any value directly in the bot — bot overrides take priority over sheet data for that field.
How to get a live CSV URL from Google Sheets
1
Open the sheet · click the specific tab you want
Make sure you're on the tab (e.g. "Expense Log") — not just the sheet.
2
File → Share → Publish to web
A dialog opens. First dropdown: select the tab name. Second dropdown: select Comma-separated values (.csv). Hit Publish.
3
Copy the URL it gives you · paste below
URL looks like: https://docs.google.com/spreadsheets/d/...pub?gid=...&output=csv
This URL stays live — whenever your sheet updates, this bot can re-fetch it.

Deploy & Maintain
Host it · Share it · Update it · Keep it running
⚡
This is your operations manual for the CFO Bot. Everything you need to host it, share it with Mindy, update it, and keep it running long-term — all in one place. Do these steps once and you're set.
Step 1 · Host it on GitHub Pages (free)
✓
Create a free GitHub account
Go to github.com → Sign up. Use your [email protected] Google Workspace email — you can sign in with Google.
✓
Create a new repository called cfo-command
Click the + icon → New repository. Name it cfo-command. Set it to Public (required for free GitHub Pages). Click Create repository.
✓
Upload this HTML file
In your new repo, click "Add file" → "Upload files". Drag in this cfo-bot-v5.html file. Rename it to index.html before uploading — that's what GitHub Pages serves as the homepage. Commit the file.
✓
Enable GitHub Pages
In your repo, go to Settings → Pages. Under "Branch", select main and / (root). Click Save. In ~60 seconds your URL is live at:
https://[your-github-username].github.io/cfo-command
✓
Share the URL with Mindy
That URL works for anyone — no login required to view. Bookmark it in Chrome. Add it to your Google Workspace shared bookmark or a Notion page.
Step 2 · Add Firebase so edits sync between Phil & Mindy
i
Without Firebase: you and Mindy see the same page but edits don't sync — if Phil adds a client, Mindy won't see it until the file is re-uploaded. Firebase fixes this in real time. Free tier handles your scale forever.
✓
Go to console.firebase.google.com
Sign in with your Google Workspace account. Click "Create a project" → name it burrows-cfo → Continue → disable Google Analytics (not needed) → Create project.
2
Add a web app + get your config
In Firebase console, click the </> (Web) icon → Register app → name it cfo-command. Firebase shows you a config block. Copy it — it looks like this:
const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "burrows-cfo.firebaseapp.com", projectId: "burrows-cfo", databaseURL: "https://burrows-cfo-default-rtdb.firebaseio.com" };
3
Enable Realtime Database
In Firebase console → Build → Realtime Database → Create database. Choose "Start in test mode" (you'll lock it down later). Click Enable.
4
Paste your config into this file — then bring it back to Claude
Open cfo-bot-v5.html in any text editor. Find the FIREBASE_CONFIG block at the top (line ~12). Replace the empty values with your real config. Save. Then come back to this Claude conversation and say "Firebase config added — wire it up" and I'll activate the real-time sync layer in the code.
Step 3 · Add Google login (so you know who's editing)
1
Enable Google Auth in Firebase
Firebase console → Build → Authentication → Get started → Sign-in method → Google → Enable → add your support email → Save.
2
Tell Claude: "activate Google login"
Once Firebase config is in place, come back here and I'll add the Google sign-in flow. Phil and Mindy sign in with their @burrowsadvisory.com Google accounts. Edits are tagged with who made them.
Step 4 · How to update this bot going forward
A
For data changes (new client, new number, new expense)
Do it directly in the bot — use the + buttons, inline edits, and forms. That's what they're for. Once Firebase is live, changes sync immediately for both partners.
B
For structural changes (new section, new field, layout)
Come back to this Claude conversation (bookmark it). Describe what you want: "Add an invoicing tab" or "add a contractor hours breakdown" — I'll update the code. Then re-upload to GitHub Pages (drag and drop, 30 seconds).
C
For bigger changes (new bot, Lead Gen Bot, etc.)
Use Claude Code — open your terminal, run claude, and describe what you want built. Claude Code has direct file access so it edits your files in place and you don't have to re-upload manually.
D
Keep this conversation as your "codebase"
This Claude.ai conversation has the full history of every version. Bookmark it at claude.ai. Any time you need changes, open it and continue from where we left off — I have full context on the whole project.
